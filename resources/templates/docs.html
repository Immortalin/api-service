Availability Check

To find out the availabile options for delivery to a certain location. Requires an active user session. 

[GET] ${base-url}v1/availability
Header Description
Authorization (Basic Auth) Use your API Key as the username, and a User Auth Token as the password.

Parameter Description Values Default
lat Latitude of desired delivery location A number (e.g., -118.27367287)
lng Longitude of desired delivery location A number (e.g., 34.053892)
vehicle_id ID of vehicle A string
Keys in red are required.

Example GET request and JSON response: (show/hide) 

curl --request GET '${base-url}v1/availability?lat=34.031464&lng=-118.491809&vehicle_id=4clhMV0ewUMDB7O4460R' -u 123APIKEY:456USERAUTHKEY
JSON Response: 
{
   "success":true,
   "availability":{
      "time_limit_choices":[
         {
            "fee":599,
            "text":"within 1 hour ($5.99)",
            "time":60
         },
         {
            "fee":0,
            "text":"within 3 hours (3 left)",
            "time":180
         }
      ],
      "gallon_choices":[
         7.5,
         10,
         15
      ],
      "octane":"91",
      "gas_price":339,
      "tire_pressure_fillup_price":700
   }
}

Request Order

To make an order request given certain parameters. Usually this endpoint is used after an Availability Check so that a user has been presented with the options available at their location. 

[POST] ${base-url}v1/orders/request
Header Description
Authorization (Basic Auth) Use your API Key as the username, and a User Auth Token as the password.

Parameter Description Values Default
lat Latitude of desired delivery location A number (e.g., -118.27367287)
lng Longitude of desired delivery location A number (e.g., 34.053892)
vehicle_id ID of vehicle A string
time_limit Number of minutes in which vehicle is available to be serviced (from now) A number (chosen from the Availability)
gallons Number of gallons A number (chosen from the Availability)
gas_price Price per gallons in cents (retrieved from Availability)
(this ensures that the gas price did not change while the user was choosing delivery options) 		A number (cents USD)
delivery_fee Price per gallons in cents (retrieved from Availability)
(this ensures that the gas price did not change while the user was choosing delivery options) 		A number (cents USD)
special_instructions Optionally include special instructions (e.g., "it's parked down the driveway") Any string
Keys in red are required.

Example POST request and JSON response: (show/hide) 

curl -X POST ${base-url}v1/orders/request -u APIKEY:USERAUTHKEY -H 'Content-Type: application/json' -d '{"lat": 34.049803,       
 "lng": -118.430745,
 "vehicle_id": "KmHDbvxQYKjcCX7EPJzM",
 "time_limit": 300,
 "gallons": 12.5,
 "gas_price": 312,
 "delivery_fee": 299,
 "special_instructions": "go down driveway"}'
JSON Response: 
{
   "success": true,
   "order_id": "OaVwEoDRdsJIQUcnySvg",
   "message": "Your order has been accepted, and a courier will be on the way soon! Please ensure that the fueling door on your gas tank is unlocked.",
   "code": "order-accepted"
}

Cancel Order

Cancel an order given its ID. You can only cancel orders that are in one of the following statuses: unassigned, assigned, accepted, or enroute. 

[POST] ${base-url}v1/orders/cancel/[order_id]
Header Description
Authorization (Basic Auth) Use your API Key as the username, and a User Auth Token as the password.
Keys in red are required.

Example POST request and JSON response: (show/hide) 

curl -X POST '${base-url}v1/orders/cancel/h738fjusye8nbds3' -u APIKEY:USERAUTHKEY
JSON Response: 
{
    "success": true
}

-- OR --

{
    "success": false,
    "message": "Sorry, it is too late for this order to be cancelled."
}

Get Orders

To get a list of a user's past and current orders along with the details of each order. 

[GET] ${base-url}v1/orders/get
Header Description
Authorization (Basic Auth) Use your API Key as the username, and a User Auth Token as the password.

Parameter Description Values Default
sort Direction in which to sort orders chronologically (time created). "asc" (oldest first) or "desc" (most recent first) "desc"
start Starting index (0-based). Use with 'limit' for pagination. An integer >= 0 0
limit How many orders to retrieve. An integer >= 0 50
vehicle_id ID of vehicle A string
Keys in red are required.

Example GET request and JSON response: (show/hide)
